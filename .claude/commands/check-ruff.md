# Ruff コード品質チェック

Python コードの静的解析とフォーマットチェックを実行します。

## 実行内容

1. **Ruff リントチェック**
   - `ruff check .` でソースコードの静的解析を実行
   - PEP 8 違反、潜在的なバグ、コードスメルを検出
   - エラーや警告があれば詳細を表示

2. **Ruff フォーマットチェック**
   - `ruff format --check .` でフォーマットの確認
   - コードスタイルの一貫性をチェック
   - フォーマットが必要な場合は修正提案

3. **テストファイルのチェック**
   - `tests/` ディレクトリが存在する場合はテストコードもチェック
   - テスト品質の確保

4. **結果レポート**
   - チェック結果の要約を表示
   - 問題がある場合の修正内容を提示

5. **問題箇所の修正（問題が残っている場合）**
   - 問題のある箇所を修正
   - 修正が完了したら 1 から繰り返し実施する

## 使用場面

- コードレビュー前の品質チェック
- プルリクエスト作成前の事前チェック
- 定期的なコード品質監視
- 開発中の継続的品質管理

## 修正方法

問題が検出された場合：

- `ruff check --fix .` で自動修正可能な問題を修正
- `ruff format .` でコードフォーマットを適用
- 手動修正が必要な問題は個別に対応を実施

## 開発用エイリアス（devcontainer内で利用可能）

- `ruff-check`: リントチェック実行
- `ruff-fix`: 自動修正可能な問題を修正
- `ruff-format`: コードフォーマット適用
